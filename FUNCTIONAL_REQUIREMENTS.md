# 敬拜 PPT 制作器 · 软件功能需求文档 (FRD)

版本：v1.0　　日期：2026-07-01　　对应代码库：[hohoyo234/New-PPT](https://github.com/hohoyo234/New-PPT)

## 1. 文档说明

本文档基于当前代码库（`src/`、`supabase_*.sql`、`README.md`）反向梳理，描述系统的**已实现**功能范围、行为规则与数据结构，作为需求基线，供后续迭代、测试用例设计与代码评审参考。

## 2. 产品概述

| 项目 | 说明 |
|---|---|
| 产品名称 | 敬拜 PPT 制作器 · Worship PPT Maker |
| 产品定位 | 面向教会敬拜团队的歌词 PPT 排版生成工具 |
| 部署形态 | 纯前端静态站点（React 19 + Vite 6），零必需后端；接入 Supabase 后解锁账户/云同步/社区功能 |
| 核心价值 | 把「歌名/歌词 → 排好版的 .pptx」的过程压缩到几步，且免费背景图、免费 AI 配图、无需登录即可用 |
| 技术栈 | React 19、Vite 6、Tailwind v4、pptxgenjs、pinyin-pro、JSZip、Supabase（Auth + Postgres） |

## 3. 用户角色

| 角色 | 说明 | 权限差异 |
|---|---|---|
| 游客（未登录） | 默认身份 | 可用全部制作/歌库/导出功能；数据只存本机 `localStorage`；不可云同步、不可贡献社区精修版 |
| 注册用户 | 邮箱+密码登录（Supabase Auth） | 额外获得：个人歌库云同步（跨设备/清缓存不丢）、向社区精修库贡献歌曲 |
| 匿名访客（追踪意义上） | 每设备一个 `anon_id`（UUID，存 localStorage） | 用于使用数据统计，非登录概念 |

## 4. 总体功能架构

系统顶层分三个模式（Tab 切换，状态存于 `localStorage: ppt_mode`），共享同一顶部导航（Logo + 模式切换 + 账户入口）：

1. **Auto 模式**（自动/引导式）— 面向"每周主日敬拜"场景，分步引导。
2. **Manual 模式**（手动/自由编辑）— 面向精细调整场景，三栏工作台。
3. **Library 模式**（歌库管理）— 歌曲数据库的增删改查、备份、社区共享。

## 5. 功能需求详述

### 5.1 Auto 模式（自动生成向导）

三步流程，步骤状态机：`count → entries → confirm`。

**Step 1 — 选择歌曲数量**
- FR-A1：提供 1–6 首的数字快捷按钮，及 1–20 首范围内的自定义输入框。
- FR-A2：点击"开始"进入下一步，按数量生成对应数量的空输入框。

**Step 2 — 逐条粘贴歌曲线索**
- FR-A3：每行输入框接受任意形式的线索：纯歌名、一句歌词、`歌名 / 制作人`、`Artist - Title`。
- FR-A4：输入框聚焦且非空时，实时下拉展示歌库模糊搜索建议（`searchLibraryMulti`），点击可直接填入标准歌名。
- FR-A5：显示当前歌库统计（总数 / 有歌词数）作为参考。
- FR-A6：点击"识别并下一步"触发批量识别（见 5.1.1）。

**Step 3 — 确认、编辑与导出（单页完成）**
- FR-A7：对每首歌展示一张卡片：背景缩略图 + 页数标记、歌名/英文名/歌词/翻译输入框、"命中歌库/需补歌词"状态标签、"自动分主歌副歌"按钮、实时幻灯片预览条（可点击放大进入编辑弹窗）。
- FR-A8：背景可"随机"（从预设图池瞬时选取）、"AI 重新生成"（Pollinations 关键词生图，较慢）、或从预设网格选取。
- FR-A9：导出设置区：文件名、是否含歌名封面页、合并导出 1 个 PPT 或 ZIP 打包分开导出、统一背景或各自背景、幻灯片尺寸 16:9 / 4:3。
- FR-A10：点击导出前，若存在无歌词的歌曲，需二次确认（该曲仅生成封面页）。
- FR-A11：导出成功后，把每首曲目（含最终使用的背景）写回本地歌库，供下次识别命中。
- FR-A12：支持从确认页生成可打印的 A4 歌词单（见 5.4）。

#### 5.1.1 自动识别规则
- FR-A13：对每条输入，先整句、再解析出的标题分别去本地歌库模糊匹配（`searchLibrary`），匹配分数 ≥60 视为命中。
- FR-A14：命中后自动执行"主歌/副歌自动分段"（`detectChorus`，见 5.1.2），并复用该歌曲上次使用的背景；未命中则依据标题/歌词关键词推导背景（`deriveBackground`，见 5.1.3），并标记"需补歌词"。
- FR-A15：线索行解析规则（`parseEntryLine`）：优先识别 `Artist - Title`（英文短横线格式），再识别 `歌名 / 制作人`（斜杠/逗号分隔），否则整句视为歌名（长度<40字符）。

#### 5.1.2 主歌/副歌自动分段（`detectChorus`）
- FR-A16：仅对已用空行分隔为 3 段以上、且尚无 `[标记]` 的歌词生效。
- FR-A17：统计各"段落文本"（≥2行）重复次数，取重复次数最高（≥2次）的段落为副歌：首次出现处保留完整歌词并加 `[副歌]` 标记，后续重复处仅保留 `[副歌]` 引用行。
- FR-A18：无重复段落时不做任何改动，允许用户手动加 `[标记]`。

#### 5.1.3 背景自动推导（`deriveBackground`）
- FR-A19：按标题+歌词关键词匹配六类预设主题（光/荣耀、水/江河海、山、十字架/宝血、夜/星、平安/同在），命中即用对应预设图。
- FR-A20：无关键词命中时，退化为以歌名为主题的 AI 生成图（Pollinations，免费无需 Key）。

### 5.2 Manual 模式（手动编辑工作台）

三栏布局：歌单/编辑区（左）、实时预览（中）、样式设置（右）。

**歌单管理**
- FR-M1：支持"空白加歌"新增歌曲；支持从歌库搜索并加入歌单（`searchLibraryMulti`）；歌单最少保留 1 首（删除最后一首自动补空白曲）。
- FR-M2：歌单、当前编辑内容、样式设置、自定义背景均持久化到 `localStorage`（键 `worship_ppt_maker_v1`），刷新页面不丢失。

**歌曲内容编辑**
- FR-M3：字段包括歌名、英文名/副标题、歌词（每行一句，空行=换页）、翻译/对照歌词（按行对应）。
- FR-M4：支持繁/简双语切换（顶部"繁/简"按钮），繁体为主字段，简体存于 `*Sc` 影子字段，切换时读写对应字段；此偏好存于 `localStorage: lib_lang`，与歌库模式共享。
- FR-M5：支持 `[副歌]` 标记重复段落，只写一次自动展开（同 5.1.2 后半部分逻辑，见 5.5）。

**实时预览**
- FR-M6：按当前设置（每页行数、字号、颜色、阴影、拼音开关）逐页渲染 16:9 缩放预览，视觉效果与最终 .pptx 一致（容器查询单位换算，1pt ≈ 1/7.2 cqw）。

**背景设置**
- FR-M7：提供内置预设背景网格（纯色 + 远程图 + 本地打包图，见 5.6）+ 已上传/AI 生成的自定义背景，点击选中。
- FR-M8：支持本地图片上传（转 base64 内嵌，立即可用）。
- FR-M9：支持 AI 生成背景：输入主题词 → Pollinations 生图（模拟 1.8s 延时反馈）。
- FR-M10："整套统一背景"开关：开启时所有歌曲共用一个背景；关闭时可为每首单独指定。

**排版与文字设置**
- FR-M11：幻灯片尺寸 16:9 / 4:3 切换。
- FR-M12：每页行数（1–6）、歌词字号（20–72pt）、翻译字号（12–48pt）滑杆。
- FR-M13：是否包含歌名封面页开关；是否显示拼音开关（显示于歌词上方，汉语拼音带声调符号，来自 `pinyin-pro`）。
- FR-M14：歌词颜色、翻译颜色取色器；文字阴影开关及三档强度（淡/中/浓）。

**导出与歌词单**
- FR-M15：点击"生成 PPT"按当前设置为全部歌曲生成合并 .pptx 并下载，文件名 `Worship_<歌单名>_<日期>.pptx`。
- FR-M16：导出成功后把每首歌（含背景）写回本地歌库。
- FR-M17：支持生成可打印歌词单（同 5.4），文案随当前繁/简语言切换。

### 5.3 Library 模式（歌库管理）

- FR-L1：列表展示全部歌曲：序号、（繁/简可切换）标题、英文名、作曲/制作人、调性标签、"有歌词/仅歌名"状态标签。
- FR-L2：支持模糊搜索：自动过滤语气词/填充词（如"那个""好像""什么什么"等）与首尾助词后按分词匹配，标题命中优先，支持"记不全歌词也能搜到"的容错。
- FR-L3：三档过滤：全部 / 有歌词 / 仅歌名。
- FR-L4：新增（空白）、编辑（含曲名繁简、英文名、作曲/作词/歌手/出版/调性、歌词繁简、英文歌词/对照）、删除（二次确认）。
- FR-L5：导出整库为 JSON 备份文件；导入 JSON 按标题合并（已存在则更新、否则新增），返回新增/更新计数。
- FR-L6：云同步按钮（登录用户可用）：双向同步个人云端库，返回"上传 N 首、拉取 N 首"提示（详见 5.7）。
- FR-L7：搜索时若命中"社区精修版"（他人已贡献且未在本地库中的同名曲目），单独分区展示，点击"查看精修内容"需先通过《共享互助协议》授权（首次弹窗，之后不再提示），确认后写入本地歌库。
- FR-L8：每条曲目可"贡献至社区精修库"：需登录；歌词为空时拒绝并提示先补全歌词；首次贡献需通过《内容合规协议》授权（之后不再提示）。

### 5.4 歌词单打印（Lyric Sheet）

- FR-P1：Auto / Manual 模式均可一键生成 A4 打印页（新窗口 `window.open`），若被浏览器拦截则提示"请允许弹出窗口"。
- FR-P2：内容为选中歌曲的展开后歌词（`[副歌]` 等标记先展开，见 5.5），按顺序编号排版，中文行下方可选显示对照翻译（斜体、缩小字号）。
- FR-P3：页面内置控制条：字号滑杆（9–30pt）、分栏（1栏/2栏）、显示/隐藏翻译开关、"打印/存为 PDF"按钮（触发浏览器打印，控制条本身在打印时隐藏）。

### 5.5 段落展开引擎（`expandSongSections`，跨模式共用）

- FR-E1：仅当歌词中存在独立一行的 `[名称]` 标记时启用，否则原样返回。
- FR-E2：首次出现的 `[名称]` 及其后续到下一个空行/标记前的歌词，定义该段落内容；此后任何单独出现、下方无内容的同名 `[名称]` 行，视为引用，自动替换为该段落已保存的完整歌词。
- FR-E3：英文/翻译行按对应的中文行顺序同步展开，保证中英文行不错位。
- FR-E4：展开结果同时用于：屏幕预览、导出 .pptx 分页、打印歌词单——三处视觉/内容强一致。

### 5.6 分页规则（`paginateLyrics`，跨模式共用）

- FR-D1：若歌词中含空行分隔的段落，则每个空行分隔的段落固定成一页（用户可通过空行精确控制每页行数）。
- FR-D2：若无空行分隔，则按设置的"每页行数"（Auto 固定 2 行；Manual 可调 1–6 行）自动分页。
- FR-D3：（预留）支持按每页显式指定行数数组 `pageLines`，数组用尽后重复最后一个值。
- FR-D4：翻译行按序与中文行一一对应配对，翻译行数少于中文行数时空缺处不显示翻译。

### 5.7 颜色与可读性自适应（`resolveSlideColors`）

- FR-C1：背景为图片时，遵循用户设定的文字颜色，但强制在图片上叠加半透明黑色遮罩（透明度 55%）以保证可读性。
- FR-C2：背景为纯色时：若该色相对亮度 > 0.6（浅色），强制使用深色文字（`#111111` / `#374151`）防止文字消失；否则使用用户设定颜色。
- FR-C3：文字阴影三档强度（淡/中/浓）在预览（CSS text-shadow）与导出（pptxgenjs outer shadow）中使用等效参数，保证所见即所得。

### 5.8 背景来源

| 来源 | 说明 |
|---|---|
| 预设纯色 | 8 种固定色值（森林深绿、午夜星空等） |
| 预设远程图 | Unsplash 图库精选（圣洁光芒/宁静时刻/福音之光/群山呼唤/圣灵如水） |
| 本地打包图 | `src/assets/backgrounds/*.{jpeg,jpg,png}`，Vite 构建期以 glob 方式打包，标签由文件名自动美化生成 |
| 用户上传 | 任意本地图片，读取为 base64 内嵌，立即可用且可直接进入导出 |
| AI 生成 | 调用 Pollinations 免费图像 API（`image.pollinations.ai`），拼接敬拜风格提示词，无需 API Key，16:9、1920×1080 |

- FR-B1：所有背景来源在预览与 .pptx 导出中使用同一套图片，保证一致性。
- FR-B2：导出时会预抓取每张远程/AI 背景图为 base64 后再嵌入 .pptx；单张抓取超时（30s）或失败时，该曲位置的背景退化为纯色，并在导出结果中标记 `bgEmbedFailed`，前端提示"部分背景图无法加载，已用纯色代替（文件已导出）"。

### 5.9 PPTX 导出引擎（`pptxgenjs`）

- FR-X1：支持两种页面尺寸：16:9（13.333"×7.5"，对应 1920×1080）、4:3（10"×7.5"）；设计基准坐标系 10×5.625，按比例缩放到实际页面尺寸，保证不同尺寸下版式一致。
- FR-X2：单曲页面结构：（可选）多曲汇编时的过渡"曲名大字报"页 → 歌名封面页（可选）→ N 张歌词页。
- FR-X3：歌词页逐行渲染：可选拼音行 → 中文歌词行（加粗）→ 可选翻译行（斜体），整体垂直居中于页面。
- FR-X4：字体统一使用 Microsoft YaHei（中文兼容性最佳）；字号在 12–72pt（歌词）、10–48pt（翻译）范围内按用户设置换算并限幅。
- FR-X5：合并导出（`exportMerged`）生成单一 .pptx，文件名 `Worship_<名称>_<日期>.pptx`。
- FR-X6：分开导出（`exportZip`）为每首歌单独生成 .pptx，用 JSZip 打包为 `Worship_<名称>_<日期>.zip`，文件名做安全化处理（过滤非法字符、限长 60、重名自动加序号）。

### 5.10 账户与认证

- FR-U1：基于 Supabase Auth 的邮箱+密码注册/登录/登出；未开启邮箱验证时登录后立即生效，开启验证时提示"请查收验证邮件"。
- FR-U2：常见错误信息本地化为中文友好提示（密码错误、邮箱已注册、密码过短、邮箱格式错误、频率限制等）。
- FR-U3：游客可用全部制作类功能；仅云同步与社区贡献功能要求登录，未登录时点击相关按钮会弹出登录框。

### 5.11 云同步（个人私有库）

- FR-S1：登录用户的歌库变更（新增/编辑/删除）实时写入 Supabase 表 `ppt_user_songs`，以 `(user_id, title)` 为唯一键 upsert。
- FR-S2：RLS 策略保证用户只能读写自己的行，实现租户隔离（换设备、清浏览器缓存不丢失个人数据）。
- FR-S3：手动"同步"操作双向合并：拉取云端全部记录合并进本地（仅新增/更新，不删除本地未上传的条目、不删除内置曲库），并把本地尚未上传的非内置曲目推送上云；完成后提示"上传 N 首、拉取 N 首"。

### 5.12 社区共享库

- FR-G1：共享表 `ppt_song_library` 对所有人可读；写入（新增/更新/删除）仅限已登录用户（详见 5.13 安全需求）。
- FR-G2：搜索歌库时（≥2字符，350ms 防抖）后台并行查询共享库，仅展示被标记 `is_curated=true` 且本地库中尚无同名曲目的结果，标注"社区精修版"。
- FR-G3：贡献流程：用户点击"贡献至社区精修库" → 校验已登录、已有歌词 → 首次需通过《内容合规协议》勾选确认 → 按标题 upsert 至共享表并置 `is_curated=true`，记录 `contributed_by`。
- FR-G4：查看流程：用户点击"查看精修内容" → 首次需通过《共享互助协议》勾选确认 → 该曲写入本地歌库供后续使用。
- FR-G5：两类协议的同意状态各自独立记录于 `localStorage`（键含版本号），同意后不再重复弹出。

### 5.13 合规与协议

- FR-T1：顶部账户区可随时查看《用户服务协议 / 版权声明》只读页（无需勾选，点击"我知道了"关闭）。
- FR-T2：协议内容涵盖：服务性质（技术排版工具，不分发版权作品）、内容版权归属用户/原作者、用户自负侵权责任、通知—删除（避风港）机制、隐私声明（个人歌库仅本人可见，不出售数据）、使用统计披露、社区内容非营利声明。
- FR-T3：《用户服务协议》页内嵌"允许匿名使用统计"开关，与追踪模块的opt-out状态双向同步。

### 5.14 使用数据追踪

- FR-K1：无痕匿名标识：每设备生成 `anon_id`（UUID，`localStorage`）；每次页面加载生成 `session_id`（UUID，`sessionStorage`）。
- FR-K2：事件类型：`session_start`、`session_end`（含停留秒数及"未导出"标记）、`mode`（切换模式）、`export`（导出，含歌曲数/合并方式/文件名）、`search`（搜索防抖后的查询词+结果数，含"无结果"标记）、`click`（关键按钮点击，如打印歌词单/贡献社区/上传背景/AI背景/随机背景）、`error`（导出失败等）。
- FR-K3：双写：Supabase 表 `ppt_usage_events`（持久可查询）+ Google Sheet（经 Apps Script Web App Webhook，可选，URL 未配置则跳过）；任一 sink 缺失不影响应用可用性（fire-and-forget，失败静默吞掉）。
- FR-K4：登录用户事件附带邮箱，游客仅附带匿名 ID；不采集其他 PII。
- FR-K5：用户可在《用户服务协议》弹窗中一键关闭统计（写 `localStorage` opt-out 标记），关闭后两个 sink 均不再发送。
- FR-K6：页面隐藏/卸载时用 `sendBeacon` 保证 `session_end` 事件尽量送达。

## 7. 数据模型（Supabase）

### 7.1 `ppt_song_library`（共享曲库 / 社区精修库）
- 任何人可读；仅登录用户可写（新增/更新/删除）。
- 关键字段：`title`（唯一索引）、`english_title`、`producer`、`lyrics`、`english_lyrics`、`bg`（JSON）、`is_curated`（是否精修）、`contributed_by`、`updated_at`。

### 7.2 `ppt_user_songs`（个人私有云库）
- RLS：仅 `user_id = auth.uid()` 可读写自己的行。
- 唯一键：`(user_id, title)`。
- 字段较共享库更丰富：含 `title_sc`、`composer`、`lyricist`、`singer`、`publication`、`key`、`lyrics_sc`、`bg`、`settings`（预留）等。

### 7.3 `ppt_usage_events`（使用行为事件）
- 字段：`ts`、`session_id`、`user_email`、`anon_id`、`type`、`mode`、`detail`、`duration_sec`、`device`、`browser`、`lang`、`referrer`。

## 8. 已知边界与限制（当前实现现状）
